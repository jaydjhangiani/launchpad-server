# Bloomberg Launchpad – Indian Markets
### Ruliad Capital Management Systems

A real-time, Bloomberg-style equity monitoring terminal for **NSE-listed Indian stocks**.  
Built with Flask (Python) + yfinance + pure HTML/CSS/JS. No React, no database, no paid APIs.

## Overview

The app reads `~625` NSE stock symbols from an Excel workbook (`blp work.xlsx`), distributes them across **20 sector panels** in a 5-column × 4-row grid, and streams live prices from Yahoo Finance every **30 seconds**. The UI auto-refreshes in the browser every **5 seconds** without a full page reload.

Key capabilities:

| Feature | Detail |
|---|---|
| Live NSE prices | via `yfinance` (Yahoo Finance backend) |
| 20 sector panels | Brokerages, Banks, IT, Pharma, Metals … |
| Sort | Click any column header (SYM / LAST / CHG% / VOL) |
| Add ticker | `+ ADD` button in each panel header |
| Remove ticker | Hover a row → click `×` |
| Persistence | Add/remove changes survive server restarts |
| Price cache | Last-known prices reload instantly on restart |
| Indices | NIFTY 50 / SENSEX / BANK NIFTY in the header bar |

## Quick Start

### Prerequisites

```powershell
# Python 3.10+  (check with: python --version)
pip install flask yfinance pandas openpyxl
```

### Run the server

```powershell
cd C:\Dev\launchpad_app
python app.py
```

Open your browser at **http://localhost:5000**

- Prices from the last run appear **within 5 seconds** (loaded from `price_cache.json`).  
- A full live refresh of all 625 symbols takes **60–90 seconds** on first start.  
- The header shows `DATA: Xs AGO` so you always know how fresh the prices are.

## Architecture

```
┌───────────────────────────────────────────────────────┐
│  Browser (http://localhost:5000)                      │
│                                                       │
│   index.html  ──poll every 5s──▶  GET /api/panels    │
│                                                       │
│   JS: initGrid() on first load                        │
│       updatePanel() on every poll                     │
│       sort, add, remove all via fetch()               │
└──────────────────────┬────────────────────────────────┘
                       │ JSON
┌──────────────────────▼────────────────────────────────┐
│  Flask  (app.py)                                      │
│                                                       │
│   GET  /              → renders index.html (no-cache) │
│   GET  /api/panels    → returns all 20 panels + prices│
│   POST /api/refresh   → triggers immediate re-fetch   │
│   POST /api/panel/<n>/add    → add ticker to panel n  │
│   POST /api/panel/<n>/remove → remove ticker          │
└──────────────────────┬────────────────────────────────┘
                       │
┌──────────────────────▼────────────────────────────────┐
│  Background Thread  (background_updater)              │
│                                                       │
│  Startup:  _load_price_cache()   (disk → memory)      │
│            update_all_prices()   (live fetch)         │
│  Loop:     sleep(30s) → update_all_prices()           │
│                                                       │
│  yfinance.download() in batches of 50 symbols         │
│  Results stored in price_cache dict (thread-safe)     │
│  Results saved to price_cache.json after every cycle  │
└───────────────────────────────────────────────────────┘
```

### Threading model

- One daemon thread runs `background_updater()` for the entire app lifetime.  
- A `threading.Lock` (`cache_lock`) protects all reads and writes to `price_cache` and `indices_cache`.  
- Flask routes read the cache under the lock; the background thread writes under the lock.  
- Setting `use_reloader=False` is **mandatory** — otherwise Flask would spawn two background threads.

## Configuration Reference

All tuneable constants live near the top of `app.py`:

| Constant | Default | Description |
|---|---|---|
| `FETCH_INTERVAL` | `30` | Seconds between background price refreshes |
| `BATCH_SIZE` | `50` | Symbols per `yfinance.download()` call |
| `CUSTOMIZATIONS_FILE` | `"user_customizations.json"` | Path to add/remove persistence file |
| `PRICE_CACHE_FILE` | `"price_cache.json"` | Path to disk price cache |

**To change the refresh rate** edit `FETCH_INTERVAL` in `app.py`:

```python
FETCH_INTERVAL = 60   # refresh once per minute instead
```

**To increase stocks per panel** edit the slice in the panel-building loop:

```python
panels.append({"sector": sector_name, "stocks": panel_stocks[:15]})
#                                                              ^^^
#                                                      change 15 to any number
```

## How Prices Work

### Symbol convention
All stocks are from NSE. When querying Yahoo Finance, the `.NS` suffix is appended automatically by the code (e.g. `RELIANCE` → `RELIANCE.NS`). You never need to type `.NS` anywhere in the UI or config files.

### Price refresh lifecycle

```
Server start
    │
    ├─ 1. _load_price_cache()          loads price_cache.json → memory
    │      → prices available within ~1s (stale but valid)
    │
    ├─ 2. update_all_prices() begins   first live fetch (60-90s total)
    │      → fetch in batches of 50, 0.3s delay between batches
    │      → after each batch, prices are updated in memory
    │
    ├─ 3. price_cache.json saved       after full cycle completes
    │
    └─ 4. Every 30s thereafter         repeats step 2 & 3
```

### How the UI stays current
Every 5 seconds the browser calls `GET /api/panels`. It compares the new prices to the last known prices and applies CSS flash animations (green flash = price up, red flash = price down). The `DATA: Xs AGO` indicator in the top-right shows seconds since the last full backend refresh.

### Stocks showing `--`
About 80 of the 625 symbols in the 2017-era Excel workbook are delisted or renamed. These show `--` in every column. They are harmless. To clean them up, delete them via the UI or `user_customizations.json` (see below).

## API Reference

All endpoints return JSON. Base URL: `http://localhost:5000`

---

### `GET /api/panels`

Returns all 20 panels with live prices.

**Response shape:**

```json
{
  "panels": [
    {
      "sector": "Brokerages",
      "stocks": [
        {
          "symbol":     "EDELWEISS",
          "name":       "Edelweiss Financial Services",
          "price":      109.41,
          "change":     0.30,
          "change_pct": 0.27,
          "volume":     1234567
        }
      ]
    }
  ],
  "indices": {
    "NIFTY 50":   { "price": 22345.60, "change": -45.20, "change_pct": -0.20 },
    "SENSEX":     { "price": 73456.10, "change": -112.0, "change_pct": -0.15 },
    "BANK NIFTY": { "price": 47890.00, "change": 120.50, "change_pct":  0.25 }
  },
  "timestamp":    "10:32:15 IST",
  "date":         "08 Mar 2026",
  "fetch_age":    12,
  "next_refresh": 18
}
```

`fetch_age` = seconds since the last completed background refresh.  
`next_refresh` = seconds until the next scheduled refresh.

---

### `POST /api/refresh`

Triggers an immediate background price refresh (non-blocking).

```bash
curl -X POST http://localhost:5000/api/refresh
```

Response: `{"status": "refresh triggered"}`

---

### `POST /api/panel/<panel_index>/add`

Adds a ticker to a panel. The panel index is 0-based (see table above).

```bash
curl -X POST http://localhost:5000/api/panel/5/add \
     -H "Content-Type: application/json" \
     -d '{"symbol": "HDFCBANK"}'
```

Response (success):
```json
{ "status": "ok", "symbol": "HDFCBANK", "price": 1782.50, "change_pct": 0.34 }
```

Response (already exists):
```json
{ "error": "HDFCBANK is already in this panel" }   ← HTTP 409
```

---

### `POST /api/panel/<panel_index>/remove`

Removes a ticker from a panel.

```bash
curl -X POST http://localhost:5000/api/panel/5/remove \
     -H "Content-Type: application/json" \
     -d '{"symbol": "HDFCBANK"}'
```

Response: `{ "status": "ok", "removed": "HDFCBANK" }`

## Adding / Removing Tickers

### Method A — Via the UI (recommended)

**To ADD a ticker:**

1. Locate the panel you want (e.g. "Private Banks").
2. Click the **`+ ADD`** button in the panel's top-right corner.
3. A text box appears. Type the **NSE symbol** exactly (case-insensitive), e.g. `HDFCBANK`.
4. Press **Enter** or click **ADD**.
5. The ticker is appended to the panel immediately. Price populates on the next poll (within 5s).
6. The change is saved to `user_customizations.json` automatically — it **survives server restarts**.

> **Finding the right symbol:** Use the NSE website (nseindia.com) or search `<company name> NSE symbol`. Do NOT add `.NS` — the app does this automatically.

**To REMOVE a ticker:**

1. Hover your mouse over any stock row in a panel.
2. A red **`×`** button appears on the right side of the row.
3. Click **`×`**.
4. The row fades out and disappears.
5. Removal is saved to `user_customizations.json` automatically.

---

### Method B — Via the REST API (scripting / bulk operations)

See [API Reference](#api-reference) above. Use `curl`, Postman, or a Python script.

**Python example — add multiple tickers to a panel:**

```python
import requests

panel_index = 1  # PSU Banks
symbols_to_add = ["INDIANB", "BANKINDIA", "UNIONBANK"]

for sym in symbols_to_add:
    r = requests.post(
        f"http://localhost:5000/api/panel/{panel_index}/add",
        json={"symbol": sym}
    )
    print(sym, r.json())
```

---

### Method C — Directly editing `user_customizations.json`

This file is checked on every server startup and live-applied.  
Format:

```json
{
  "0": {
    "added": [
      { "symbol": "ICICIPRULI", "name": "ICICI Prudential Life Insurance" }
    ],
    "removed": ["MOTILALOFS"]
  },
  "5": {
    "added": [
      { "symbol": "HDFCBANK", "name": "HDFC Bank" }
    ],
    "removed": []
  }
}
```

- Keys are **panel indices as strings** (`"0"` through `"19"`).
- `added` = list of `{symbol, name}` objects to append to the panel.
- `removed` = list of symbol strings to exclude from the panel.
- The `name` field is displayed in the UI — set it to the company's full name for clarity, or just repeat the symbol if unknown.

**After editing this file, restart the server** for changes to take effect:

```powershell
# Stop current server (Ctrl+C), then:
python app.py
```

## Editing Stocks in Source Data

The ultimate source of truth for which stocks appear in each panel is **`sheet_data.json`**.  
This was generated once from `blp work.xlsx` and is now the live reference.

### To permanently add/rename/reorganise stocks for a sector

1. Open `sheet_data.json` in any text editor.
2. Find the key for the sheet you want to edit:

```json
"Sheet 2": [
  { "symbol": "SBIN",       "name": "State Bank of India" },
  { "symbol": "BANKBARODA", "name": "Bank of Baroda" },
  ...
]
```

3. Add, remove, or rename entries. Each entry must have:
   - `"symbol"` — NSE ticker (no `.NS`).
   - `"name"` — display name shown in the panel header tooltip area.

4. Save the file and restart the server.

> Changes to `sheet_data.json` become the new baseline. `user_customizations.json` *layered on top* of it, so clean up that file too if you're reorganising.

### To re-generate `sheet_data.json` from scratch (from the Excel file)

```powershell
python extract_sheets.py
```

This reads `blp work.xlsx` and overwrites `sheet_data.json`. Run this if you update the Excel file.

## Customisation Persistence Files

### `user_customizations.json`

- Records every `+ ADD` and `× remove` action from the UI or API.
- Applied on top of `sheet_data.json` at server startup.
- **Safe to delete** — the server starts clean from `sheet_data.json` defaults.
- **Key insight:** if you remove a stock via the UI and then re-add it to `sheet_data.json`, the removal in this file will still win. Delete the entry from `removed` in this file to un-suppress it.

### `price_cache.json`

- Written by the background thread after every successful full refresh cycle.
- Loaded instantly at server startup so prices appear immediately (no 60–90s warmup).
- **Safe to delete** — the server will fetch fresh prices from Yahoo Finance on next startup (slower first load).
- **Do not manually edit** — it is a plain dict of `{ "SYMBOL": { price, change, change_pct, volume } }` but is overwritten every 30s.

## UI Controls Guide

### Header bar (top)

- **LAUNCHPAD** logo — top-left branding
- **NIFTY 50 / SENSEX / BANK NIFTY** — live index prices with colour-coded change %
- **MARKET OPEN / CLOSED** — calculated from IST 09:15–15:30, Mon–Fri
- **Clock** — current IST time (updates every second)
- **DATA: Xs AGO** — seconds since last completed backend price refresh
- **REFRESH** button — triggers an immediate background re-fetch

### Each panel widget

- **Title bar** — sector name (orange), ▲ count / ▼ count of stocks up/down today
- **`+ ADD`** button — opens input field to add a ticker (see above)
- **Column headers** — click **SYM / LAST / CHG% / VOL** to sort; click again to reverse; ↕ = unsorted, ↑/↓ = sorted direction
- **Stock rows** — symbol (yellow), name, price (white), change% (green/red/grey), volume (dim grey)
- **Hover a row** → reveals red **`×`** delete button

### Footer ticker tape

- Scrolling tape of all live prices across all 20 panels.
- Updates every 5 seconds in sync with the grid.
- Green = up today, red = down today.

## Troubleshooting

### Server won't start

```
Error: Address already in use
```
Another process is using port 5000. Kill it:
```powershell
Get-Process -Name python | Stop-Process -Force
```
or change the port in `app.py`:
```python
app.run(host="0.0.0.0", port=5001, ...)
```

---

### All prices show `--`

1. `price_cache.json` may not exist yet (first ever run).
2. Yahoo Finance may be rate-limiting — wait 2–3 minutes for the background thread to complete.
3. Check the terminal running `app.py` — look for `[HH:MM:SS] Updated X/625 prices in Ys`.

---

### A specific stock always shows `--`

The symbol is likely **delisted** or renamed on NSE.  
Verify it on: https://www.nseindia.com/  
If delisted, remove it via the UI (`×` button on hover) or from `sheet_data.json`.

---

### Added stock disappeared after server restart

- If the stock was added via the **UI**, it is in `user_customizations.json` and will persist.
- If you added it to `sheet_data.json` directly, make sure the symbol isn't also listed in the `"removed"` list inside `user_customizations.json`.

---

### Prices are stale (DATA badge shows a very large number)

- The background thread may have crashed. Restart the server.
- Yahoo Finance sometimes returns `401 Unauthorized` during heavy rate-limiting. The thread automatically retries on the next cycle (30s later).
- Check terminal output for `[WARN]` lines.

---

### Browser showing old version of the page

The server sends `Cache-Control: no-store` headers, but some browsers are aggressive.  
Hard-refresh: **Ctrl + Shift + R** (Windows/Linux) or **Cmd + Shift + R** (Mac).

## Known Limitations

| Limitation | Detail |
|---|---|
| End-of-day prices (EOD) | yfinance returns the **last trading day's closing price**, not a live streaming tick. Prices update every 30s during market hours via the background thread but are not true real-time. |
| ~80 delisted symbols | The source Excel was created in 2017. Some companies have since merged, delisted, or been renamed. |
| No intraday charts | Prices only; no OHLCV candlestick charts. |
| Single machine only | Runs on localhost. To share on a LAN, change `host="0.0.0.0"` and open port 5000 in Windows Firewall. |
| No login / auth | Any user with network access can add/remove tickers. Not intended for multi-user production use. |
| Max 15 stocks per panel at startup | Baseline limited by `panel_stocks[:15]` in app.py. Added tickers via UI have no cap. |
